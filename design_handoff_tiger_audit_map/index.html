TIGER Audit — Design Handoff

High-fidelity design reference for an alternative cartographic presentation of the four-zone TIGER/Line audit. Two working prototypes (2D + 3D), full design-token tables, and a screenshot-capture workflow. Reference-only; the intended downstream is a port into a modern application stack.

2D — Leaflet

Flat map, warm cartographer palette, low-bandwidth, prints cleanly. Class toggles, search, basemap picker, selection panel.

3D — MapLibre GL

Tilted aerial with full pan / tilt / zoom, dark control-room palette, IBM Plex typography. Imagery, OSM, and dark basemaps.

Documentation

  • README.md — architecture, design tokens (warm + dark palettes), typography, spacing scale, component specs, data model, and class-narrative copy.
  • CAPTURE-MEDIA.md — browser-console scripts for capturing the six 3D state screenshots and the 24-frame orbit-sweep GIF (the 2D screenshots are committed under screenshots/; 3D capture requires running the prototype locally).

Captured screenshots

  • 2d-01-default.png — default state, Blue Ash / Montgomery zone, all classes visible.
  • 2d-02-critical-only.png — Class AB and Class A only.
  • 2d-03-positron-basemap.png — CARTO Positron basemap.

Data

  • prototypes/data.json — 2D-map summary + features array (1,934 ways for Blue Ash / Montgomery, 5-decimal polylines, defect class + severity per feature).
  • prototypes/ways.geojson — 3D-map GeoJSON FeatureCollection with the same population, plus a rank property for line-sort-key ordering.

Status

Prototypes load and run on Pages directly via the links above. Local-server requirement noted in the original handoff (relative fetch() calls fail on file://) is satisfied by Pages serving the directory over HTTP. The preserveDrawingBuffer: true flag has been pre-applied to the 3D MapLibre constructor so the screenshot-capture script in CAPTURE-MEDIA.md works without further modification.
